## Key Changes

### Race Condition Fixes (internal/types/types.go)
- Added `active atomic.Bool` field to Order struct
- Added thread-safe methods: `IsActive()`, `Activate()`, `Deactivate()`
- Added `Snapshot()` method for safe order field reading during triggers
- Changed from mutex-based active flag to lock-free atomic operations

### OMS (internal/oms/positions.go)
- Fixed race between OnPositionUpdate and OnPriceTick
- Added `order.Deactivate()` before `pool.PutOrder()`
- Created `adjustReduceOnlyOrdersWithOB()` and `adjustCloseOnTriggerOrdersWithOB()`
- Pass orderbook directly to avoid calling getOrderBook() (would deadlock)

### OMS (internal/oms/triggerflow.go)
- `handleTrigger()` now uses `order.Snapshot()` for thread-safe field reading
- Added `createChildOrderInputFromSnapshot()` method
- `deactivateLinkedOrders()` calls `Deactivate()` before `pool.PutOrder()`
- Added backward-compatible `createChildOrderInput()` for tests

### OMS (internal/oms/validation.go)
- Added `reduceOnlyMu sync.Mutex` to protect `reduceOnlyCommitment` map
- Prevents deadlock when `updateReduceOnlyCommitment` is called during trimming

### OrderBook Tests (internal/orderbook/orderbook_bench_test.go)
- Fixed copy-lock issue: changed `taker := takerTemplate` to use pointer
- Prevents copying Order struct containing sync.RWMutex

### Thread-Safe Order Methods (internal/types/types.go)
- `Remaining()` - RLock for reading Quantity - Filled
- `StatusSnapshot()` - RLock for reading Status
- `FilledSnapshot()` - RLock for reading Filled
- `QuantitySnapshot()` - RLock for reading Quantity
- `SetStatus(status)` - Lock for writing Status
- `AddFilled(qty)` - Lock for incrementing Filled
- `SetQuantity(qty)` - Lock for writing Quantity
- `SetUpdatedAt()` - Lock for writing UpdatedAt
- `SetClosedAt()` - Lock for writing ClosedAt

### Lock Ordering (CRITICAL)
To prevent deadlock, always acquire locks in this order:
1. `posMu` - Position update lock
2. `mu` - OMS main lock
3. `ob.mu` - Orderbook lock
4. `order.mu` - Individual order lock
