## Key Changes

### OrderBook (internal/orderbook/orderbook.go)
- Added `matchSlicePool` for zero-allocation slice recycling
- Fixed benchmark to return buyer orders to pool
- Result: **-16% time, -79% memory**

### Trigger Monitor (internal/triggers/monitor.go)
- Simplified from complex tree+linked list to simple heap-based approach
- Fixed BUY traversal logic (was broken - started from max instead of min)
- Removed "flipSide" logic for CloseOnTrigger orders (misunderstanding)
- Added node pool for trigger nodes
- Result: **-44% Add, -13% Check time**

### TriggerFlow (internal/oms/triggerflow.go)
- Fixed potential deadlock: removed redundant lock in OnPriceTick
- `handlePriceTick` was already holding lock when calling `OnPriceTick`
- Removed redundant `lastPrices` update (was duplicated)

### OMS (internal/oms/service.go)
- Nested maps structure verified as correct for per-user operations
- No O(n) iterations found - all nested map usage is appropriate
